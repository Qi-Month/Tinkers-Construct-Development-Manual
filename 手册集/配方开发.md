# 目录

> 仅收录由匠魂添加的配方类型，部分配方类型由于还未弄清暂未收录

> 配方文件位于`data/{modid}/recipes/`目录下，无论路径和文件名，只要是正确的配方文件都会被识别

> 在进行开发时建议使用`JEI`进行开发,`JEI`相比于`EMI`和`REI`可以更方便的查看或复制配方ID

* **关于Recipe ID(配方ID)**
  * 配方ID往往是配方文件的所在目录,在开发中用于查看配方类型以及写法时显得尤为重要
  * 在添加`JEI`后游戏内`F3+H`开启`高级提示框`后便可查看  
  ![高级提示框](/手册集/手册插图/提示/高级提示框.png)
  * 在JEI内随便打开一个配方文件将光标放在输出上便可查看   
  ![ID查看](/手册集/手册插图/提示/ID查看.png)
  * 如上图铁锭浇筑的配方所示,根据配方ID我们可以得知配方文件在
  `.jar/data/tconstruct/recipes/smeltery/casting/metal/iron/ingot_gold_cast.json`
  * 顺带一提在`KubeJS`开发中配方ID也非常重要,往往用于删除或者覆盖原配方,如果要手抄配方ID效率低且非常容易出错,但是`JEI`给我们提供了复制配方ID的按键,我们自行配置即可(默认没有绑定任何按键,按照自己的习惯绑定即可 [我这边绑的是P])
  ![按键绑定](/手册集/手册插图/提示/按键绑定.png)

# 通用配方

## 融化

> 融化“融化物品”，输出“输出流体”。如果在熔铸炉内融化，会产生“额外流体”(如果有的话)

示例[灵浆球融化成熔岩，在熔铸炉内额外产生岩浆](/手册集/data/litter_wolf_fufu/recipes/smeltery/melting/lava_to_ichor.json)

```json
{
    "type": "tconstruct:melting",
    "ingredient": {
        "item": "tconstruct:ichor_slime_ball"
    },
    "result": {
        "amount": 250,
        "fluid": "minecraft:lava"
    },
    "temperature": 1000,
    "time": 10,
    "byproducts": [
        {
            "amount": 500,
            "fluid": "tconstruct:magma"
        }
    ]
}
```

* `type` `"tconstruct:melting"`表示这是一个融化配方
* `ingredient {}` 融化配方的“融化物品”
  * `item`或`tag`: 物品或物品标签命名空间
* `result {} ` 融化配方的“输出流体”
  * `amount` 流体的量，单位mB
  * `fluid` 流体的命名空间
* `temperature` 该融化配方需要的最低温度，炉子温度低于此温度将无法进行该配方
* `time` 融化需要的时间，单位(25/6)游戏刻，约等于0.2秒
* `byproducts [{}...]` (可选)当在铸造炉进行融化额外产生流体，可以有多种
  * `amount` 流体的量，单位mB
  * `fluid` 流体的命名空间

## 融化燃料

> “燃料”燃烧为融化炉、冶炼炉、熔铸炉提供一定时间的融化温度

```json
    "duration": 250,
    "fluid": {
        "amount": 50,
        "fluid": "tconstruct:magma"
    },
    "temperature": 300
}
```

## 铸件台浇筑

> 三种浇筑的不同
> * 空浇筑：没有“铸模”。“输入流体”浇筑在铸件台上冷却后得到“输出物品” - _没有键`cast`和`cast_consumed`_
> * 消耗铸模浇筑：“输入流体”浇筑在铸件台上的“铸件”中冷却后得到输出“输出物品”，同时“铸模”被消耗 - _键`cast_consumed`为`true`_
> * 不消耗铸模浇筑：“输入流体”浇筑在铸件台上的“铸件”中冷却后得到输出“输出物品”，保留“铸模” - _键`cast_consumed`为`false`_

示例[消耗铸模浇筑：肉汤浇鸡蛋得到鸡刷怪蛋](/手册集/data/litter_wolf_fufu/recipes/smeltery/casting/table/egg_to_chicken.json)

```json
{
    "type": "tconstruct:casting_table",
    "cast": {
        "item": "minecraft:egg"
    },
    "cast_consumed": true,
    "fluid": {
        "amount": 1000,
        "name": "tconstruct:meat_soup"
    },
    "result": "minecraft:chicken_spawn_egg",
    "cooling_time": 60
}
```

* `type` `"tconstruct:casting_table"`表示这是一个铸件台浇筑配方
* `cast {}` (可选)“铸模”。若不存在，则该配方不需要铸模参与
  * `item`或`tag` 物品或物品标签的命名空间
* `cast_consumed` (可选)是否消耗“铸模”。必须与键`cast`同时存在
* `fluid {}` “输入流体”
  * `amount` 流体的量，单位mB
  * `name`或`tag` 流体或流体标签的命名空间
* `result` 冷却后形成“输出物品”的命名空间
* `cooling_time` 冷却时间，单位tick

# 特殊配方

> 材料一支的配方简单了解：
> 
> 材料（material）, 该配方可以让特定材料在**工匠台/工匠砧**上用特定物品进行修复, 若材料的[`craftable`](./新材料开发.md#材料基础信息-definition)属性为true, 则允许特定物品在**部件制造台**上制造该材料的部件
>
> 材料浇筑（material_fluid）, 该配方可以让**冶炼炉/熔铸炉**中达到特定温度的流体在铸模中制造特定材料的部件
>
> 材料熔化（material_melting）, 该配方可以让特定材料的部件在**冶炼炉/熔铸炉**中达到特定温度时熔化为特定流体

## 材料

示例[下界之星的材料配方](/手册集/data/litter_wolf_fufu/recipes/materials/nether_star.json)

```json
{
    "type": "tconstruct:material",
    "ingredient": {
        "item": "minecraft:nether_star" 
    },
    "material": "litter_wolf_fufu:nether_star",
    "needed": 1,
    "value": 1
}
```

示例[熔岩的材料配方](/手册集/data/litter_wolf_fufu/recipes/materials/_lava_/ball.json)

```json
{
    "type": "tconstruct:material",
    "ingredient": {
        "tag": "forge:slimeball/ichor" 
    },
    "material": "litter_wolf_fufu:lava",
    "needed": 4,
    "value": 1
}
```

```json
{
    "type": "tconstruct:material",
    "ingredient": {
        "item": "tconstruct:ichor_congealed_slime" 
    },
    "material": "litter_wolf_fufu:lava",
```

```json
{
    "type": "tconstruct:material",
    "ingredient": {
        "item": "tconstruct:ichor_slime" 
    },
    "material": "litter_wolf_fufu:lava",
    "needed": 4,
    "value": 9,
    "leftover": {
        "count": 1,
        "item": "tconstruct:ichor_congealed_slime"
    }
```

* `type` `"tconstruct:material"`表示这是一个材料配方

* `ingredient {}`
  * `item`或`tag`消耗的物品或物品类命名空间

* `material` 关联的输出材料命名空间，使材料对应的工具可以消耗指定物品修复耐久。 _如果这个材料基本信息的`craftable`为`true`，则材料对应的部件可以在部件制造台上消耗指定物品制造_

* `needed` 制造或修复一个材料部件需要的(锭)的基础值。

* `value` 消耗一个物品提供的(锭)的值

* `leftover` 当一个物品提供的(锭)的值超过制造部件所需的值时溢出物品
  * `item`或`tag`溢出的物品或物品类命名空间
  * `count`溢出物品的数量

## 有序强化

> 有序强化配方是指符合条件的工具/武器/装备在**工匠砧/工匠台**上用特定物品摆出特定顺序可以获取特定强化

示例[丝绸之触有序强化配方](/手册集/data/tconstruct/recipes/tools/modifiers/ability/silky.json)

```json
{
  "type": "tconstruct:modifier",
  "allow_crystal": true,
  "check_trait_level": true,
  "inputs": [
    {
      "item": "tconstruct:silky_cloth"
    },
    {
      "item": "tconstruct:silky_cloth"
    },
    {
      "item": "tconstruct:silky_cloth"
    },
    {
      "item": "tconstruct:silky_cloth"
    },
    {
      "item": "tconstruct:silky_cloth"
    }
  ],
  "level": 1,
  "result": "tconstruct:silky",
  "slots": {
    "abilities": 1
  },
  "tools": {
    "tag": "tconstruct:modifiable/harvest"
  }
}
```

* `type` `tconstruct:modifier`声明这是一个有序强化配方

* `result` 该配方获取的强化

* `allow_crystal` 若为`true`则该强化可以在**强化调配台**中被提取成对应的强化水晶

* `check_trait_level` 若为`true`则该配方会检查被强化工具上的强化等级，一般用于当该配方应用的强化有可能出现在工具材料本身上，防止重复强化造成浪费

* `input [{}]`  强化需要的特定物品摆放
   * `item` 指定特定物品
   * `tag` 指定带有该标签的物品
   * `ingredient [{}]` 指定更多物品，内部可以使用`item`或`tag`等
     * 当使用`item`指定物品时, 可以附带`nbt: {}`来匹配物品的nbt

* `level` 当对应的值为`int`时, 该配方会直接将被强化物品的该强化等级设置为对应等级
  * `level`可以对应一个对象`{}`, 该对象允许`max`(int)和`min`(int)两个键值对
  * 当level对应为一个对象时，若等级小于`min`则会直接提升到`min`对应的值，之后逐级增加直到达到`max`
  * _注: level对应对象一般用于无序强化，这里这是提一嘴_

* `slots {}` 该强化需要消耗的槽位类型和数量，若没有这个键则该强化不消耗任何槽位
  * 槽位类型: `upgrades`升级槽, `abilities`能力槽, `defense`防御槽, `soul`灵魂槽
  * _注: `soul`灵魂槽虽然没在实际游戏中使用到,但是是已经实装的_

* `tools` 可以应用的该配方的工具需要带的tag

## 无序强化

> 无序强化配方是指符合条件的工具/武器/装备在**工匠砧/工匠台**上用特定物品摆出特定顺序可以获取特定强化

示例[锋利无序强化配方](/手册集/data/tconstruct/recipes/tools/modifiers/upgrade/sharpness_from_shard.json)

```json
{
  "type": "tconstruct:incremental_modifier",
  "allow_crystal": true,
  "amount_per_item": 1,
  "input": {
    "tag": "forge:gems/quartz"
  },
  "level": {
    "max": 5
  },
  "needed_per_level": 36,
  "result": "tconstruct:sharpness",
  "slots": {
    "upgrades": 1
  },
  "tools": {
    "tag": "tconstruct:modifiable/melee"
  }
}
``` 
> [使用石英块的配方](/手册集/data/tconstruct/recipes/tools/modifiers/upgrade/sharpness_from_block.json)

```json
{
  "type": "tconstruct:incremental_modifier",
  "allow_crystal": false,
  "amount_per_item": 4,
  "input": {
    "tag": "forge:storage_blocks/quartz"
  },
  "leftover": "minecraft:quartz",
  "level": {
    "max": 5
  },
  "needed_per_level": 36,
  "result": "tconstruct:sharpness",
  "slots": {
    "upgrades": 1
  },
  "tools": {
    "tag": "tconstruct:modifiable/melee"
  }
}
```


* `type` `tconstruct:incremental_modifier`代表这是一个无序强化配方

* `result` `allow_crystal` `slots` `level` `tools`见上文[有序强化配方](#有序强化),两者作用写法都是相同的
  
* `needed_per_level` 提升一级所需的虚拟数量（_物品数量指的是下文`amount_per_item`定义的数量而非实际意义上的物品数量_）

* `amount_per_item` 每个实际物品数量代表多少个虚拟数量

* `input {}` 允许`item`,`tag`和`ingredient [{}]`输入

* `leftover` 当输入的虚拟数量超出时，每一点虚拟数量会返还对应的物品
